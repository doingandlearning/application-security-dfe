## Incident response (dev-friendly loop)

1. <span class="fragment">Detect & confirm (is it real?)</span>
2. <span class="fragment">Triage (scope + impact + which systems)</span>
3. <span class="fragment">Contain (block requests / disable features / revoke tokens)</span>
4. <span class="fragment">Eradicate & recover (patch, rotate secrets, redeploy)</span>
5. <span class="fragment">Learn (post-incident actions so it doesn’t recur)</span>

---

## Key takeaways from the course

1. <span class="fragment">**App security matters** — Threats and risks in web and mobile; security belongs in the lifecycle.</span>
2. <span class="fragment">**Secure SDLC** — Threat modeling, risk assessment, security in CI/CD.</span>
3. <span class="fragment">**Common vulnerabilities** — OWASP Top 10, secure coding, input validation, session and data protection.</span>
4. <span class="fragment">**Testing** — SAST/DAST, ZAP, Burp, pen-testing fundamentals.</span>

---

5. <span class="fragment">**DevSecOps** — Automation, containers, cloud, incident response.</span>
6. <span class="fragment">**LLM security** — OWASP LLM Top 10, trust but verify, PII, hallucinations; safe use of Copilot and similar tools.</span>
